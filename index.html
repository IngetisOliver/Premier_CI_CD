GIT

Guide rapide — utilisation de Git

Les concepts et commandes essentiels pour utiliser Git en local et collaborer via un dépôt distant (GitHub, GitLab, etc.).

Qu'est-ce que Git ?

Git est un système de contrôle de version distribué qui enregistre les changements dans les fichiers et permet de collaborer efficacement.

  • Historique local (snapshots)
  • Branches légères pour expérimenter
  • Fusion et résolution de conflits

Installation

Installer Git sur Linux / macOS / Windows :

# macOS (Homebrew)
brew install git

# Debian / Ubuntu
sudo apt update && sudo apt install git

# Windows: installer depuis https://git-scm.com

Configuration initiale

git config --global user.name "Ton Nom"
git config --global user.email "email@exemple.com"
# Vérifier
git config --list

Créer / cloner un dépôt

# Créer un dépôt local
mkdir mon-projet && cd mon-projet
git init

# Cloner un dépôt distant
git clone https://github.com/utilisateur/projet.git

Cycle de travail courant (Workflow)

Exemple simple : modifier, ajouter, valider, pousser.

# Voir l'état
git status

# Ajouter des fichiers à l'index (staging)
git add fichier.txt
# ou tout
git add .

# Enregistrer un commit
git commit -m "Message clair expliquant le changement"

# Récupérer les derniers changements du distant et fusionner
git pull --rebase origin main

# Envoyer vos commits
git push origin main

Branches

Les branches permettent de travailler sur des fonctionnalités sans toucher à la branche principale.

# Créer et basculer sur une branche
git checkout -b feature/ma-fonction

# Lister les branches
git branch

# Fusionner la branche dans main (depuis main)
git checkout main
git merge feature/ma-fonction

# Supprimer la branche locale
git branch -d feature/ma-fonction

Résolution de conflits

Lorsqu'il y a des conflits après un merge ou pull, Git marque les fichiers concernés. Il faut les éditer, résoudre, puis :

git add fichier-resolu.txt
git commit -m "Résolution de conflit: fichier-resolu"

Trucs & bonnes pratiques

  • Faire des commits atomiques et des messages clairs.
  • Pull avant de commencer à travailler pour éviter les conflits.
  • Utiliser des branches pour chaque fonctionnalité / ticket.
  • Relire les changements avec git diff avant commit.

Cheat-sheet (commandes fréquentes)

git status
git add .
git commit -m "msg"
git push

# Branches
git checkout -b nom-branche
git checkout main
git merge nom-branche

git pull --rebase

Exemple visuel du flux collaboratif

Typiquement : fork/clone → créer une branche → PR/MR → revue → merge → déployer

# Exemple résumé
# 1. mettre à jour main
git checkout main
git pull origin main

# 2. créer une branche et y travailler
git checkout -b feat/login
# ... coder
git add .
git commit -m "feat: ajouter écran de connexion"

# 3. pousser la branche
git push -u origin feat/login

# 4. ouvrir une Pull Request sur la plateforme distante (GitHub/GitLab)
